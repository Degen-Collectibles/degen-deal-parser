Degen Collectibles Plaid Security Overview

Effective date: May 15, 2026

Scope

This overview describes security practices for the Degen Collectibles operations portal and its Plaid bank-feed integration. The integration is used for internal bookkeeping and reconciliation only.

Governance

Security and data-protection responsibilities are assigned to Degen Collectibles ownership and authorized administrators. Security issues are reviewed based on risk, affected data, required containment, and any legal or vendor notification obligations.

Access Control

  • Application access is role-based and limited to authorized staff.
  • Finance and bookkeeping pages require authenticated administrative access.
  • Access is removed when a user no longer needs it for business duties.
  • Production credentials are kept out of source code and browser console logs.

Plaid Token Handling

  • Bank connections are authorized through Plaid Link.
  • Plaid client secrets are stored in runtime environment configuration.
  • Plaid access tokens are stored in encrypted form in the application database.
  • Connections and credentials are revoked or rotated if access is no longer needed or compromise is suspected.

Data Protection

  • Production access is served over HTTPS with TLS.
  • Plaid data is used only for bookkeeping, reconciliation, and internal reporting.
  • Plaid data is not sold or shared with unauthorized third parties.
  • Sensitive financial data is restricted to personnel with a business need.

Logging and Monitoring

Sync failures, import status, reconciliation mismatches, parsing errors, and operational exceptions are visible to authorized staff through application pages or logs. The application avoids logging passwords, API secrets, or full sensitive token values.

Change and Vulnerability Management

Application changes are made in the source repository, tested where practical, and deployed through the approved hosting workflow. Changes involving authentication, financial data, token handling, webhooks, or reconciliation logic receive additional review before production use. Dependencies and hosted assets are patched based on severity and operational risk.

Incident Response

If a security incident is suspected, Degen Collectibles will contain access, preserve relevant logs, rotate affected credentials, revoke affected Plaid connections where needed, assess affected data, document corrective action, and notify affected parties, vendors, or regulators when legally required.
