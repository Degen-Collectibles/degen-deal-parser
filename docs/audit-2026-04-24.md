## 🏆 Combined Top 5 — fix these first

Synthesized across all three reports, weighted by severity × consensus.

1. **Purge / terminate flow leaks accounts via stale tokens & half-anonymizes PII** (`team_admin_employees.py:947,1002-1057`)
   Claude flagged purge as irreversible with no notify. Codex flagged "CCPA purge" as misleading (only nulls EmployeeProfile, leaves user/audit intact). Cursor flagged terminate/purge as leaving outstanding `InviteToken` + `PasswordResetToken` rows consumable — stale invites can resurrect a terminated user.
   **Fix:** inside terminate + purge transactions, bulk-stamp `used_at=utcnow()` on invite/reset tokens, finish the PII anonymization (or rename the action), and add a soft-delete/notification step.
   _(flagged by: **Claude · Codex · Cursor**)_

2. **Supply-request status has no server-side state machine** (`team_admin_supply.py:77-111`)
   Codex: any→any transitions via crafted POST. Cursor: denied→approved→ordered loops; concurrent approvals both succeed; `approved_by_user_id` only set on approve/deny, not on ordered. Claude flagged mis-clicks as irreversible (related UX).
   **Fix:** port the 409-on-invalid-transition guard from `team_admin_timeoff.py:188`; set `approved_by_user_id` on every non-submitted transition.
   _(flagged by: **Codex · Cursor** + related from Claude)_

3. **Clockify admin page: N+1 API calls + PII leak** (`team_admin_clockify.py:121-126, 235-275, 466-523, 487-493`)
   Codex: up to 100 serial Clockify API calls on page render when `include_hours=1`. Cursor: same N+1 flagged as HIGH. Claude: two full-table scans on same page. Codex also noted raw external emails exposed to anyone with `admin.employees.view`.
   **Fix:** gate `include_hours` behind an explicit button, cache responses for 60s, consolidate the two table scans into a single JOIN, and mask external emails unless the viewer has a reveal permission.
   _(flagged by: **Claude · Codex · Cursor**)_

4. **SESSION_SECRET double-duty as password salt + token HMAC** (`auth.py:49, 362-369`)
   Claude MED, Cursor HIGH + HIGH. Rotating `SESSION_SECRET` silently locks out every legacy unsalted account AND invalidates every outstanding invite/reset. This is a one-config-rotation-away outage.
   **Fix:** migrate empty salts during boot and drop the fallback branch; require `EMPLOYEE_TOKEN_HMAC_KEY` as a distinct env var; document the rotation procedure.
   _(flagged by: **Claude · Cursor**)_

5. **Employee list: `?q=` filters after slicing to 200 rows AND bulk-decrypts legal names** (`team_admin_employees.py:179, 208-221`)
   Claude PERF: `list(...).all())[:200]` is a full scan. Cursor BUG HIGH: same — filter misses matches beyond row 200. Codex SEC HIGH: the list page also bulk-decrypts draft employees' legal names, bypassing the audited reveal flow entirely.
   **Fix:** push the ILIKE filter into SQL, `.limit(200)` in SQL, and stop bulk-decrypting legal names in list/search rendering.
   _(flagged by: **Claude · Codex · Cursor**)_
